# bbb-MVaLePyPin

The bbb pinmux can be a real PITA. Tracking the pinmux register offsets so you can check/change a pin's configuration gets old pretty fast. Config-pin can be your friend, but it may not work with upstream kernel. Also, using devicet tree overlay blobs - especially when inserted by uBoot at boot time - may be a bit meh. Geert sure maintains the Very Cool (tm) config FS based device-tree overlay adding patches - but one still needs to get the device-trees done. Using the pin macros from Linux build to craft the pincontroller properties may help a bit, but having to preprocess the DTS sources is still a bit ... meh. So is finding the correct pin-names.

For years I had a TODO-item to create a debug tool for myself to help this. Never quite got to do it. Then the AI came.

This code is _very much_ AI-produced. You should use it at your own risk! NO WARRANTY GIVEN! MAY DAMAGE YOUR HARDWARE!

Anyways, here it is. The first version of pinmux-tool, which can provide you information about the BBB pins - and when ran on the BBB, it should be able to access the actual pinnmux via /dev/mem.
